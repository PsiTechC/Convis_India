PAGES CREATED

1. Main Integrations Dashboard
   Location: /convis-web/app/integrations/page.tsx
   Route: /integrations
   Features:
   - Overview of all integrations (Jira, HubSpot, Email)
   - Statistics cards (Total, Active, Inactive, Types)
   - Integration type cards with connection status
   - Quick navigation to individual integration pages
   - Filter by integration type
   - Dark mode support

2. Email Integration Page
   Location: /convis-web/app/integrations/email/page.tsx
   Route: /integrations/email
   Features:
   - Add/manage email SMTP accounts
   - Test email connections
   - Support for Gmail, Outlook, SendGrid, Amazon SES
   - Form with SMTP configuration
   - Connection test with success/failure modal
   - Delete integrations
   - Provider setup guides

3. Workflows Dashboard
   Location: /convis-web/app/workflows/page.tsx
   Route: /workflows
   Features:
   - List all workflows
   - Workflow statistics (Total, Active, Executions, Success Rate)
   - Filter by status (All, Active, Inactive)
   - Enable/disable workflows with toggle
   - Delete workflows
   - Navigate to workflow builder
   - View execution history

4. Workflow Builder (Create/Edit)
   Location: /convis-web/app/workflows/create/page.tsx
   Route: /workflows/create
   Features:
   - 4-step wizard interface
     * Step 1: Basic Info (name, description, priority)
     * Step 2: Trigger Event (call_completed, call_failed, etc.)
     * Step 3: Conditions (optional filtering)
     * Step 4: Actions (configure integrations)
   - Visual progress indicator
   - Dynamic action configuration based on integration type
   - JSON config editor for actions
   - Add multiple conditions with AND/OR logic
   - Add multiple actions per workflow
   - Error handling options (continue, stop, retry)

5. Workflow Execution Logs
   Location: /convis-web/app/workflows/executions/page.tsx
   Route: /workflows/executions
   Features:
   - View all workflow executions
   - Filter by status (All, Completed, Failed)
   - Execution details (duration, actions, errors)
   - Timestamp and trigger event info
   - Status badges (completed, failed, partial)
   - Error messages display

DATA FLOW
========================================

1. User navigates to /integrations
2. Fetches integrations from API
3. Displays cards and statistics
4. User can:
   - Add new integration
   - Test existing integration
   - Navigate to specific integration page
   - Create workflows

Workflow Creation Flow:
1. Click "Create Workflow"
2. Step 1: Enter basic info
3. Step 2: Select trigger event
4. Step 3: Add conditions (optional)
5. Step 4: Configure actions
6. Submit workflow to API
7. Redirect to workflows list
